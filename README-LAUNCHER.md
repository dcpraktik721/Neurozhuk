# README-LAUNCHER — Архитектура лаунчеров «Поймай Жука»

Документ для разработчика, учителя информатики или будущего сопровождающего проекта. Описывает, как устроены лаунчеры, как пересобрать дистрибутив и как тестировать сборку перед сдачей жюри.

> Пользовательская инструкция для школьников и родителей лежит в `КАК-ЗАПУСТИТЬ.txt` рядом. Этот файл — для тех, кто будет код поддерживать.

---

## 1. Цель лаунчеров

Игра «Поймай жука» — это Next.js-приложение. Запуск из исходников требует терминала и команд `npm install` / `npm run dev`. Целевая аудитория (школьники 8–12 лет, родители, жюри конкурса) с этим не справится.

Лаунчеры решают эту проблему: один двойной клик скрывает всю техническую часть и открывает игру в браузере.

Цели:

- Один файл — один клик — игра запущена.
- Минимум зависимостей: только Node.js LTS (18+).
- Кроссплатформенность: Windows / macOS / Linux.
- Идемпотентность: повторный запуск не ломает установку.
- Понятные сообщения на русском при ошибках.

## 2. Структура файлов

```
neurozhuk/
├── Запустить-игру.bat          ← Windows-лаунчер (cmd.exe)
├── Запустить-игру.command      ← macOS-лаунчер (bash, двойной клик в Finder)
├── Запустить-игру.sh           ← Linux-лаунчер (bash, через терминал)
├── scripts/
│   ├── launcher-common.sh      ← Общая логика для macOS/Linux
│   └── launcher-common.bat     ← Общая логика для Windows
├── КАК-ЗАПУСТИТЬ.txt           ← Пользовательская инструкция (UTF-8 BOM)
├── README-LAUNCHER.md          ← Этот файл
├── package.json                ← Зависимости и скрипты Next.js
├── package-lock.json           ← Точные версии (ОБЯЗАТЕЛЬНО в zip)
├── .env.local                  ← Переменные окружения (ОБЯЗАТЕЛЬНО в zip)
├── next.config.ts              ← Конфиг Next.js
├── src/                        ← Исходный код игры
└── public/                     ← Картинки, звуки, шрифты
```

### Что делает каждый лаунчер

| Файл | Платформа | Что делает |
|------|-----------|------------|
| `Запустить-игру.bat` | Windows | Делегирует `scripts\launcher-common.bat`, ставит UTF-8 кодировку, открывает чёрное окно |
| `Запустить-игру.command` | macOS | Через shebang `#!/bin/bash` вызывает `scripts/launcher-common.sh`. Открывается двойным кликом в Finder |
| `Запустить-игру.sh` | Linux | Тонкая обёртка над `scripts/launcher-common.sh`, требует `chmod +x` |
| `scripts/launcher-common.sh` | macOS+Linux | Общая логика: проверка node, npm install при необходимости, npm run dev, открыть браузер |
| `scripts/launcher-common.bat` | Windows | То же самое, но на cmd: `where node`, условный `npm install`, `start "" http://localhost:3000`, `npm run dev` |

## 3. Поток запуска

```
┌──────────────────────────────────────────────────────────┐
│  Пользователь                                            │
│       │ двойной клик по «Запустить-игру.{bat|command|sh}»│
│       ▼                                                  │
│  Платформенный лаунчер                                   │
│       │ chcp 65001 / cd "$(dirname "$0")"                │
│       ▼                                                  │
│  scripts/launcher-common.{sh|bat}                        │
│       │                                                  │
│       ├─► [1] Проверить наличие node                     │
│       │       нет → красное сообщение «установите Node»  │
│       │       есть → дальше                              │
│       │                                                  │
│       ├─► [2] Проверить node_modules/                    │
│       │       нет → npm install (2–3 минуты)             │
│       │       есть → пропустить                          │
│       │                                                  │
│       ├─► [3] Открыть http://localhost:3000 в браузере   │
│       │       (с задержкой ~3 сек, чтобы dev успел       │
│       │       подняться)                                 │
│       │                                                  │
│       └─► [4] npm run dev                                │
│              ▼                                           │
│       Next.js dev server на :3000                        │
│              ▼                                           │
│       Игра в браузере 🎮                                 │
└──────────────────────────────────────────────────────────┘
```

Закрытие: `Ctrl+C` в терминале или закрытие окна — Next.js dev server завершается, порт освобождается.

## 4. Как пересобрать zip-дистрибутив

### Что обязательно входит в zip

- Все исходники: `src/`, `public/`, `package.json`, `package-lock.json`, `next.config.ts`, `tsconfig.json`, `eslint.config.mjs`, `postcss.config.mjs`
- Все лаунчеры: `Запустить-игру.bat`, `Запустить-игру.command`, `Запустить-игру.sh`
- Папка `scripts/` целиком
- `КАК-ЗАПУСТИТЬ.txt`
- `README-LAUNCHER.md` (этот файл)
- `.env.local` — иначе игра упадёт на старте, проверьте что там нет секретов до сборки

### Что НЕ должно попасть в zip

- `node_modules/` — установится при первом запуске (~150 МБ, бессмысленно тащить)
- `.next/` — кэш сборки, перегенерируется
- `.git/` — история разработки, ~50 МБ, не нужна жюри
- `.DS_Store`, `Thumbs.db` — мусор ОС
- `.claude/`, `.code-review-graph/` — служебные папки разработки

### Команда сборки (macOS/Linux)

```bash
cd /Users/macbook15/Downloads/ЛЕВ/neurozhuk
zip -r ../Поймай Жука-игра.zip . \
  -x "node_modules/*" \
  -x ".next/*" \
  -x ".git/*" \
  -x ".DS_Store" \
  -x ".claude/*" \
  -x ".code-review-graph/*" \
  -x "*.log"
```

Проверка получившегося архива:

```bash
unzip -l ../Поймай Жука-игра.zip | head -30
unzip -l ../Поймай Жука-игра.zip | wc -l   # должно быть ~250-400 файлов
```

Целевой размер архива: **5–15 МБ** (без `node_modules/`). Если получился >50 МБ — что-то лишнее попало в архив.

### Проверка после распаковки

```bash
mkdir /tmp/neurozhuk-test && cd /tmp/neurozhuk-test
unzip ~/Downloads/Поймай Жука-игра.zip
ls -la                       # должны быть все 3 лаунчера и КАК-ЗАПУСТИТЬ.txt
file КАК-ЗАПУСТИТЬ.txt       # должно быть UTF-8 with BOM
```

## 5. Известные проблемы и решения (FAQ для разработчика)

### macOS Gatekeeper

`.command`-файл не подписан Apple Developer ID — при первом запуске Gatekeeper блокирует. Это нормально для школьного проекта.

**Обход для пользователя:** правый клик → «Открыть» → в диалоге снова «Открыть».

**Обход для разработчика на своей машине:**

```bash
xattr -d com.apple.quarantine Запустить-игру.command
```

После распаковки zip атрибут карантина появится снова — поэтому в `КАК-ЗАПУСТИТЬ.txt` всегда показываем правый клик.

### Кодировки на Windows

Стандартная кодовая страница `cmd.exe` — `cp866`, поэтому кириллица в эхе ломается. В `Запустить-игру.bat` и `scripts/launcher-common.bat` ОБЯЗАТЕЛЬНО первой строкой:

```bat
@chcp 65001 >nul
```

Сами `.bat`-файлы должны быть сохранены в **UTF-8 с BOM** — иначе Windows читает их как cp1251 и крякозябры остаются. Проверка:

```bash
file scripts/launcher-common.bat
# должно быть: UTF-8 Unicode (with BOM) text
```

### Кириллица в путях

Если пользователь распаковал игру в путь с кириллицей (например `C:\Users\Лев\Рабочий стол\Поймай Жука`), `npm install` обычно справляется, но некоторые пакеты с native-биндингами могут падать. Решения:

- В лаунчере не использовать `cd` через переменные с пробелами без кавычек.
- Везде писать `cd /d "%~dp0"` (Windows) и `cd "$(dirname "$0")"` (Unix).
- В `КАК-ЗАПУСТИТЬ.txt` рекомендовать распаковку на Рабочий стол (короткий путь).

### Порт 3000 занят

Next.js при занятом порту автоматически предлагает 3001/3002 и пишет это в консоль. Но `start http://localhost:3000` уже открылся. Лаунчер не должен предугадывать порт — пусть пользователь увидит правильный URL в окне терминала. В FAQ инструкции это упомянуто.

### `npm install` зависает за прокси

В школах часто прокси/файрвол режет npm registry. Рабочих универсальных решений нет, но можно подсказать:

```bash
npm config set registry https://registry.npmmirror.com/
```

В стандартный лаунчер это НЕ зашиваем — только в README для учителя.

## 6. Чек-лист тестирования перед сдачей

Тестировать ОБЯЗАТЕЛЬНО на чистой машине (или новом пользователе ОС), без `node_modules/` рядом.

### Windows 10/11

- [ ] Скачать zip с GitHub/Я.Диска
- [ ] Распаковать на Рабочий стол (путь содержит кириллицу — это норма)
- [ ] Двойной клик по `Запустить-игру.bat`
- [ ] SmartScreen → «Подробнее» → «Выполнить в любом случае»
- [ ] Кириллица в окне читаема
- [ ] `npm install` отработал без ошибок (≤ 5 мин)
- [ ] Браузер открылся на http://localhost:3000
- [ ] Игра загрузилась, жуки бегают, очки начисляются
- [ ] Закрытие окна корректно завершает Next.js
- [ ] Повторный запуск идёт уже без `npm install`

### macOS (Intel + Apple Silicon)

- [ ] Распаковать архив через двойной клик в Finder
- [ ] Двойной клик по `Запустить-игру.command` → Gatekeeper
- [ ] Правый клик → «Открыть» → «Открыть» — игра запустилась
- [ ] Терминал показывает шаги установки на русском
- [ ] Браузер открылся автоматически
- [ ] Игра играбельна
- [ ] `Ctrl+C` в Терминале корректно завершает

### Linux (Ubuntu 22.04+)

- [ ] `unzip Поймай Жука-игра.zip`
- [ ] `cd Поймай Жука-игра && chmod +x Запустить-игру.sh`
- [ ] `./Запустить-игру.sh`
- [ ] Игра запустилась, браузер открылся (или показано «откройте http://localhost:3000»)

### Кросс-проверки

- [ ] Без интернета: при наличии `node_modules/` игра должна запускаться
- [ ] Без Node.js: лаунчер показывает понятное сообщение со ссылкой на nodejs.org
- [ ] Порт 3000 занят: показывается предупреждение, не падает молча
