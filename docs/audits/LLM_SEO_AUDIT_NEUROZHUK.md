# LLM SEO Audit: neurozhuk.ru

Дата проверки: 2026-05-03
Режим проверки: `puppeteer-mcp-server` + live HTML/HTTP audit
Проверенный URL: `https://neurozhuk.ru/`

## Executive Summary

Сайт **технически доступен для LLM-ботов**: главная страница отдается в SSR/SSG-HTML, основной текст, `H1`, `H2`, CTA и смысловые блоки присутствуют в исходном коде страницы **без необходимости исполнять клиентский JavaScript**.

Главный технический риск не в JavaScript-рендеринге, а в **неполной структуре для knowledge extraction**:

1. на главной странице **нет JSON-LD structured data**;
2. `H1` слишком общий и не содержит точного entity+offer;
3. есть конкретные цифры, но **не хватает подтвержденных фактов, ссылок на исследования и табличной фактологии**;
4. отсутствует `llms.txt`;
5. hero-секция содержит много декоративного SVG/animation markup, что увеличивает шум в исходном HTML.

Итог:
**ChatGPT/Claude/Perplexity смогут прочитать страницу**, но **вероятность рекомендации и точного цитирования можно заметно повысить** за счет структурированных данных, более прямого H1/lead-блока и усиления фактической конкретики.

## Шаг 1. Техническая доступность страницы для LLM-ботов

### 1.1. Зависит ли основной контент от клиентского JavaScript

**Вывод:** нет, критически не зависит.

Что подтверждено:

- live-страница отвечает `HTTP/2 200`;
- в ответе есть `x-nextjs-prerender: 1`;
- исходный HTML содержит:
  - `H1`
  - `H2`
  - основной вводный абзац
  - CTA-кнопки
  - смысловые секции
  - футер и навигацию

Это означает, что страница **не является пустым клиентским shell**, который LLM-бот увидит как белый лист.

### 1.2. Что найдено в исходном HTML

В исходном HTML присутствуют:

- `title`: `Поймай Жука — Тренируй мозг играючи`
- `meta description`
- `canonical`
- Open Graph / Twitter metadata
- `H1`: `Тренируй мозг играючи`
- `H2`-секции:
  - `Что развивает Поймай Жука`
  - `Три шага к результату`
  - `Тренировки, подкреплённые наукой`
  - `Для детей и взрослых`
  - `Отслеживай свой прогресс`
  - `Почему Поймай Жука`
  - `Готовы начать тренировку?`

Также в HTML есть первый смысловой абзац:

`Поймай Жука — онлайн-платформа для развития внимания, скорости мышления и когнитивных навыков. Для детей и взрослых.`

### 1.3. Вердикт по JavaScript-риску

**Вердикт:**
бот ChatGPT/Claude/Perplexity **сможет прочитать страницу без рендеринга сложных скриптов**.

Почему:

- текст уже есть в SSR HTML;
- основная структура страницы не скрыта за hydration-only UI;
- страница не требует выполнения JS для получения ключевых текстовых сущностей.

### 1.4. Но есть технический минус

Несмотря на SSR, HTML перегружен декоративным содержимым:

- inline SVG
- animation markup
- React Flight payload
- крупные hero-декорации

Размер главной страницы в HTML: около `123692` байт.

Это **не блокирует индексирование**, но:

- добавляет шум;
- ухудшает соотношение “полезный текст / служебная разметка”;
- может снижать эффективность извлечения сущностей у некоторых ботов.

### 1.5. Критическая ошибка уровня Step 1

#### C1. На главной нет structured data, хотя SEO-слой в коде уже подготовлен

Факт:

- `application/ld+json` на главной странице отсутствует;
- при этом заготовки схем уже есть в [src/lib/seo.ts](/Users/macbook15/Downloads/ЛЕВ/neurozhuk/src/lib/seo.ts:1).

Почему это плохо для LLM:

- LLM-моделям проще извлекать сущности, когда есть формально размеченные:
  - `Organization`
  - `WebSite`
  - `SoftwareApplication` / `WebApplication`
  - `FAQPage`

Сейчас этот слой на главной не используется.

## Шаг 2. Структура текста и уровень “воды”

### 2.1. Есть ли прямой ответ на интент пользователя в первом абзаце

**Частично да.**

Сильная сторона:

- первый абзац прямой и понятный:
  `Поймай Жука — онлайн-платформа для развития внимания, скорости мышления и когнитивных навыков. Для детей и взрослых.`

Это хороший LLM-friendly lead.

Слабая сторона:

- главный `H1` слишком общий: `Тренируй мозг играючи`
- он не фиксирует:
  - название продукта
  - тип продукта
  - предмет области
  - целевую аудиторию

Для LLM и поисковых систем лучше, если `H1` будет самодостаточным.

### 2.2. Насколько текст разбит на чанки

**В целом хорошо.**

Что есть:

- 1 `H1`
- 7 `H2`
- 20 `H3`
- секции логически разделены
- тексты в большинстве блоков короткие
- каждая карточка отвечает за одну тему

Это хороший формат для LLM chunking.

### 2.3. Где есть лишняя “вода”

Критической водянистости нет, но есть мягкие маркетинговые формулировки без опоры на факты:

- `Комплексный подход к тренировке когнитивных функций через увлекательную игровую механику`
- `Поймай Жука адаптируется под любой возраст и уровень подготовки`
- `Тренировки, подкреплённые наукой`

Проблема не в длине фраз, а в том, что они **обещают больше, чем доказывают прямо на странице**.

Для LLM это снижает цитируемость: модель видит тезис, но не видит опору.

### 2.4. Критическая ошибка уровня Step 2

#### C2. `H1` не соответствует entity-first логике для LLM

Текущее состояние:

- `H1`: `Тренируй мозг играючи`

Почему это плохо:

- это slogan, а не сущностный заголовок;
- LLM хуже понимает, что именно за продукт перед ней;
- при цитировании модель скорее возьмет первый абзац, чем headline.

Что нужно:

Заменить headline на формулу вида:

- `Поймай Жука — онлайн-тренировка внимания и арифметики для детей и взрослых`
- или `Поймай Жука — когнитивные тренировки и математические игры онлайн`

## Шаг 3. Структурированные данные, списки, таблицы, факты

### 3.1. Есть ли списки и таблицы

На странице есть:

- маркированные списки / list-like блоки: да
- таблицы: нет

Для обычного SEO это допустимо. Для LLM SEO это слабее, потому что таблицы и компактные fact blocks проще парсятся и цитируются.

### 3.2. Есть ли конкретные цифры

Да, на странице есть цифры:

- `1000+ упражнений`
- `10 уровней мастерства`
- `5 минут в день`
- `Для детей 7–14 лет`
- `540`
- `7`
- `92%`
- `30 секунд` в CTA-блоке

Это плюс.

### 3.3. Достаточно ли фактов для цитирования

**Недостаточно.**

Проблема:

- цифры есть, но не всегда понятно, что они означают и на чем основаны;
- утверждение `Тренировки, подкреплённые наукой` не сопровождается:
  - именами исследований,
  - авторами,
  - годами,
  - ссылками на статьи,
  - хотя бы конкретными научными тезисами.

LLM лучше цитируют страницы, где есть формулировки такого типа:

- `Рабочая память участвует в удержании и обработке информации при решении задач`
- `Скорость обработки информации связана с эффективностью выполнения когнитивных задач`
- `Тренировка занимает 5 минут в день и включает упражнения на внимание, устный счет и скорость реакции`

То есть нужны **операциональные факты**, а не только общие обещания.

### 3.4. Критическая ошибка уровня Step 3

#### C3. На главной нет JSON-LD structured data

Это уже подтверждено фактически:

- live HTML не содержит `application/ld+json`
- в коде schema helpers есть, но не подключены к home page

#### C4. Нет `llms.txt`

Проверка:

- `https://neurozhuk.ru/llms.txt` -> `404`

Это не стандартный SEO-блокер, но для LLM-доступности полезно иметь:

- краткое описание проекта
- важные URL
- preferred summaries
- canonical facts

#### C5. Недостаточно доказательной конкретики в научном блоке

Текущий блок говорит, что тренировки “основаны на исследованиях”, но не дает:

- источников
- имен направлений
- конкретных тезисов
- ссылок на материалы

Для LLM это слабый claim.

## Итоговый технический вердикт

### Что хорошо

- главная страница индексируема без client-side rendering
- основной контент присутствует в HTML
- есть `title`, `description`, `canonical`, `OG`, `Twitter`
- есть `robots.txt`
- есть `sitemap.xml`
- структура секций и чанков в целом хорошая
- есть числовые факты и возрастная сегментация

### Что мешает LLM-рекомендациям

- на главной нет JSON-LD
- `H1` не entity-first
- научный блок недостаточно доказательный
- нет таблиц / компактных fact blocks
- нет `llms.txt`
- исходный HTML перегружен декоративным SVG и служебной разметкой

### Финальный вердикт

**Сайт уже читаем LLM-моделями.**
**Но рекомендоваться и цитироваться он будет не на максимуме**, потому что:

- сущности размечены неформально;
- facts layer слабее, чем нужен для knowledge extraction;
- главная страница пока больше похожа на качественный marketing/product page, чем на идеально LLM-optimized answer page.

## Короткий финальный ответ

Да, `https://neurozhuk.ru/` уже **может успешно читаться нейросетями**, потому что главная страница серверно рендерится и не превращается для бота в пустой JavaScript-shell.

Но чтобы сайт **чаще рекомендовали и точнее цитировали** ChatGPT, Claude и Perplexity, нужно усилить:

- entity-first headline,
- structured data на главной,
- фактологический слой,
- `llms.txt`,
- компактные LLM-friendly блоки с конкретикой.
